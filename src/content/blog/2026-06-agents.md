Ever since Google I/O 2026, I’ve been eager to try out the new stuff, but the tool that I’m writing about today is [Antigravity 2.0](https://io.google/2026/explore/pa-keynote-2). I had experimented with the Antigravity IDE before it became its separate application, but I abandoned it because the auto-complete was way too *aggressive*. So, when the new Antigravity 2.0 came out, I thought that it would be a good time to start on a side project that I had been wishing for for a long time. It’s called [aVow](https://github.com/olusheki/avow), and basically, it’s a productivity app that lets you make binding vows (unbreakable contracts) with your phone so that you’re not using it when you’re not supposed to.

At first, I was treating it like a typical chatbot. Before I do my projects, I always do a TON of research to make sure that I’m taking a sensible path that won’t lead to suboptimal results. So before I actually started with the agent, I had predetermined prompts for every step of the project that were supposed to be foolproof, assuming that the agent made no mistakes. This was the first mistake: **never assume the agent won’t make mistakes**. I was on module 4/7 for making the MVP of my app, and prior to that, everything was going right. Holding its hand, checking its work at every stage, making sure it was implementing tests, everything! Then when I tested it again, the build failed. Then I gave it the error information to pinpoint exactly what to fix, and then the build failed again. At this point, I was in a very familiar loop: You’re talking back and forth with an untrustworthy agent, who leaves trails of mistakes then confidently tells you, “Okay, everything’s fixed! It should work now!” 

That’s when I got the idea: I can actually make an **adversarial agent** whose entire goal is to scrutinize and stress test what my engineer agent is doing. And so, I told the agent to check the engineer’s work, and it found shortcuts in the tests, security flaws, and more. But I didn’t realize that this was only level 1 of utilizing the power of Antigravity 2.0.

After completing my MVP, I put it on my phone to use in real life for a week, so I can see the things I wish it had, the things that feel clunky, and what actually breaks when people use it. After two weeks, I had several grievances, but I didn’t want to deal with the friction of working back and forth with an agent on problems it won’t fix unless you catch them. So I fully utilized Antigravity 2.0’s ability to deploy subagents in plaintext, telling them each to dissect every individual grievance I had to make a comprehensive plan—one for UI, one for testing, one for architecture, etc. I did a couple of iterations on this plan, since it got some stuff wrong (this is why humans in the loop are always necessary), then when I got to the final plan, I did something different. I told the agent to split the plan across 1 agent to work on each task, to manage and check up on them continuously, and then deploy another adversarial subagent to check their work and make them iterate on mistakes. **All autonomously**. 

I sent my AI manager off, and it worked for **an HOUR STRAIGHT**. Which is INSANE, since my MVP took ~18 hrs with my original back-and-forth prompting from research to MVP. All I was doing was accepting terminal commands. When it was done, I put it on my phone again to do more IRL testing, and the rest will be filling in the cracks in the code like before. TLDR: Agentic AI 12x’d my shipping speed.
